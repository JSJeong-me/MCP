# MCP_Server



## 📝 MCP (Model Context Protocol) 사용법 정리

---

### 1. MCP의 개념 및 필요성 💡

* **MCP란?**
    * AI 모델(LLM)과 **외부 도구(Tool)**를 연결하기 위한 **표준 프로토콜**입니다.
    * 2024년 11월, Anthropic(Claude 개발사)에서 표준을 만들어 대중화되었습니다.
    * **비유:** 다양한 기기와 연결되는 **USB-C**와 같습니다. 도구가 MCP 표준에 맞게 만들어지면 AI와 쉽게 연결할 수 있습니다.
* **왜 필요한가?**
    * **LLM의 한계 극복:** LLM은 학습된 데이터만으로 다음 단어를 예측할 뿐, 실시간 데이터나 특정 기능 수행을 할 수 없습니다 (**할루시네이션** 발생).
    * **도구 연동의 표준화:** 기존에는 각각의 **API**를 AI가 사용하기 쉽도록 개별적으로 개발하여 연동해야 했으나, MCP는 이 과정을 표준화하여 **AI와 외부 도구의 연결을 쉽게** 만듭니다.

---

### 2. MCP 활용 사례 (블렌더 예시) 🖼️

* **원리:** 3D 제작 도구인 **블렌더(Blender)**가 MCP 표준에 맞게 구현되어 있다면, AI 클라이언트(예: Claude)에 블렌더 MCP를 **연결**할 수 있습니다.
* **결과:** AI는 블렌더의 사용법을 이해하고, 사용자의 요청(예: "이 이미지를 기반으로 3D 오브젝트를 만들어줘")에 따라 MCP를 통해 **직접 블렌더 프로그램을 조작**하여 작업을 수행할 수 있습니다. 

---

### 3. MCP 클라이언트 연결 및 사용 방법 (Context7 예시) ⚙️

MCP를 연결하고 사용할 수 있는 AI 환경을 **MCP 클라이언트**라고 합니다 (예: **Claude 데스크톱 버전**, **Cursor**, **VS Code**).

#### A. MCP 클라이언트 준비 (Claude 데스크톱)

1.  **Claude 데스크톱 버전**을 다운로드하고 설치합니다.
2.  Claude 앱에서 설정 메뉴 → **개발자 탭**으로 이동합니다.
3.  **로컬 MCP 서버**와 연결할 수 있는 설정 창을 확인합니다.

#### B. 기존 MCP 연결 (Context7)

1.  **Smithery**와 같은 MCP 리스트 제공 사이트에서 원하는 MCP(예: **Context7**)를 선택합니다.
2.  **Context7**은 개발 문서를 AI가 쉽게 참고할 수 있도록 연결해 주는 MCP입니다.
3.  Context7의 **JSON 명령어**를 복사합니다.
4.  Claude 데스크톱의 설정에서 **구성 편집**을 눌러 설정 파일(e.g., `claudedesktopconfig.json`)을 열고 복사한 JSON 명령어를 붙여넣어 서버를 구성합니다.
5.  Claude를 재시작하면 Context7 MCP가 클라이언트에 추가됩니다.

#### C. MCP 사용

* 채팅창에서 MCP가 제공하는 기능을 사용할 수 있습니다.
* **예시 프롬프트:** "Next.js 프레임워크로 새로운 웹사이트를 만들고 싶은데, **Context7 MCP를 이용해서** 새로운 앱 만드는 명령어를 찾아와줘."
* AI는 MCP를 호출하여 **Context7**을 통해 Next.js의 **최신 공식 문서**를 가져와 참고하여 정확한 답변을 제공합니다.

### 4. MCP 직접 만들기 (FastMCP 라이브러리) 👨‍💻

* **도구:** Python의 **FastMCP** 라이브러리 등을 사용하여 간단하게 MCP를 만들 수 있습니다.
* **구현:**
    * 라이브러리를 불러옵니다.
    * 원하는 기능을 구현한 함수에 `@mcp.tool` 등의 **어노테이션**을 추가합니다.
    * **예시:** 두 숫자를 더하는 `add_two_numbers(a, b)` 함수를 만들고 서버로 실행합니다.
* **연결 방식:**
    * **STDIO (Standard Input/Output):** 로컬 컴퓨터에서 프로그램을 실행하고 클라이언트와 소통하는 방식 (로컬용).
    * **Streamable HTTP:** 웹 서비스처럼 HTTP를 이용하여 연결하고 소통하는 방식 (원격/배포용).
* **원격 배포:** 만든 파이썬 서버 파일을 웹 서버에 **디플로이**하고, 해당 서버 주소로 클라이언트(예: VS Code)와 연결하면 누구나 어디서든 MCP를 사용할 수 있습니다.

---

### 5. MCP의 쉬운 공유 및 활용 (Porter AI) 🤝

* **문제점:** MCP를 만들어도 일반 팀원에게 VS Code 등 개발 도구를 설치하고 복잡한 MCP 설정을 하도록 요구하기 어렵습니다.
* **해결책 (Porter AI):**
    * 일반적인 ChatGPT 화면처럼 웹사이트에서 누구나 쉽게 접근하여 AI를 활용할 수 있는 플랫폼입니다.
    * **강점:** MCP 연결 환경을 미리 세팅하고 **팀원들에게 쉽게 공유**할 수 있습니다.
    * **예시 활용:**
        * **노션 MCP:** 회사 내부 규정이 담긴 노션과 연결하여 "외근 식비는 얼마까지 쓸 수 있어?"와 같은 질문에 **실제 규정**을 기반으로 답변합니다.
        * **슬랙 연동:** 업무용 슬랙 채널에서 Porter AI를 태그하여 회사 내부 DB와 연결된 MCP에 질문하면 슬랙 안에서 바로 정확한 답변을 받을 수 있습니다.
